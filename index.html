Interaktives CSS-Dashboard – UX/Frontend-Konzept

Ziel: Ein Single-File-fähiges Lern- & Experimentier-Dashboard, das CSS-Eigenschaften und Direktiven für Anfänger bis Fortgeschrittene didaktisch aufbereitet und in Echtzeit visualisiert.

1. Zielgruppe & Lernziele

Zielgruppe

  • Studierende & Quereinsteiger:innen, die CSS-Grundlagen lernen möchten.
  • Junior-Frontend-Entwickler:innen, die ihr CSS-Mentalmodell schärfen wollen.
  • Fortgeschrittene Professionals, die seltene Properties testen & dokumentieren.

Lernziele

  • Wirkung & Zusammenspiel gängiger CSS-Properties verstehen.
  • Kaskade, Vererbung, Spezifität & Quellreihenfolge anhand interaktiver Beispiele begreifen.
  • Best Practices durch sofort kopierbare Snippets anwenden.
  • Fehlerquellen identifizieren & Alternativen finden (z. B. fehlende Voraussetzungen).

6. Interaktion & UX-Details

Direkte Manipulation

  • Drag-Handles in Vorschau für Margin/Padding (Snapping 4px / 0.25rem).
  • Farbrad + eyedropper (wenn verfügbar).
  • Einheiten-Dropdown (px, rem, %, vw/vh, fr, deg).
  • Slider mit konfigurierbaren Steps & Constraints.

Produktivität

  • Undo/Redo (Ctrl/Cmd+Z/Y).
  • Code kopieren (Ctrl/Cmd+C).
  • Kurzhilfe (F1) mit Tastenkürzel-Overlay.
  • Suche (/) fokussiert Sidebar-Input.
  • LocalStorage: Session-Persistenz, Custom Presets.
Snapping & Constraints: Validierung im PropertyModel verhindert unsinnige Werte, UIControls bieten Quick-Buttons für Standardwerte (0, auto, 100%).

7. Technische Leitplanken

Architektur (Modules)

  • PropertyModel: JSON-Definitionen, Validierung, Abhängigkeiten.
  • StateStore: zentraler Zustand, History, Undo/Redo-Stack.
  • UIControls: Rendert Controls aus Schema, Event-Bindings.
  • CodeGen: Generiert CSS (Inline, Scoped, Global) + Formatierung.
  • PreviewSandbox: Shadow DOM oder iFrame, Synchronisation, Overlays.
  • Inspector: computed styles, Specificity-Berechnung, Konflikt-Hinweise.

Technik & Performance

  • Reines HTML/CSS/JS (ES6 Modules), keine Bundler.
  • Batch-DOM-Updates via requestAnimationFrame.
  • Eingaben mit Debounce, differenzielle Updates.
  • i18n: Texte in JSON (de/en), UI reagiert live auf Sprachwechsel.
  • A11y: ARIA-Labels, Fokus-Traps, High-Contrast-Mode, Respekt vor prefers-reduced-motion.

9. State-Diagramm & Event-Flow

State-Diagramm (vereinfacht)

┌──────────────┐   User Input   ┌──────────────┐
│ UIControls   │ ─────────────▶ │ StateStore   │
└──────┬───────┘                └──────┬───────┘
       │                              │
       │ Render Updates               │ Notify
       ▼                              ▼
┌──────────────┐   Generate CSS  ┌──────────────┐
│ Preview      │ ◀────────────── │ CodeGen      │
└──────┬───────┘                 └──────┬───────┘
       │ Inspector Feedback            │ History
       ▼                               ▼
┌──────────────┐                 ┌──────────────┐
│ Inspector    │◀───────────────▶│ Undo/Redo    │
└──────────────┘                 └──────────────┘

Event-Flow (Schritte)

  1. Nutzer:in ändert Control → UIControls validiert Input (PropertyModel).
  2. StateStore aktualisiert Zustand, schreibt History-Eintrag.
  3. CodeGen erzeugt differenzielles CSS (gewählte Output-Variante).
  4. PreviewSandbox injiziert Styles, Inspector misst Wirkung.
  5. Inspector liefert Konflikt-/Hinweis-Events zurück (Overlay, Toast).
  6. Toolbar & Sidebar aktualisieren „geänderte Eigenschaften“ & Preset-Status.

11. Didaktische Hinweise je Kategorie

Layout

  • Merksatz: Layout-Properties wirken oft nur bei passenden Display-Typen.
  • Übung: Positioniere ein Badge absolut und setze position: relative am Container.
  • Warnung: z-index benötigt Stacking Context (z. B. Positionierung).

Flexbox

  • Merksatz: „Flex-Kinder hören nur, wenn Eltern flexen.“
  • Übung: Erzeuge zentriertes Layout (Preset Flex-Center).
  • Hinweis: flex-basis überschreibt width.

Grid

  • Merksatz: Tracks zuerst planen, dann Areas benennen.
  • Übung: Baue 2-Spalten-Grid mit repeat & minmax.
  • Warnung: justify-content vs. justify-items unterscheiden.

Typografie

  • Merksatz: line-height ohne Einheit bevorzugen (Skalierung).
  • Übung: Stelle responsive Typo mit clamp() ein.
  • Warnung: font-size beeinflusst Rem-basierte Abstände.

Farbe & Hintergrund

  • Merksatz: Mehrere Background-Layer werden von oben nach unten gerendert.
  • Übung: Kombiniere Verlauf + Bild mit background-blend-mode Pro.
  • Warnung: Transparente Farben beeinflussen Kontrast (A11y-Check).

Dekoration & Effekte

  • Merksatz: Shadows nutzen rgba für weiche Kanten.
  • Übung: Card-Schatten-Preset anwenden & variieren.
  • Warnung: Viele Filter können Performance kosten.

Responsive & Logik

  • Merksatz: Mobile First – min-width Media Queries bevorzugen.
  • Übung: Container Query aktivieren und Komponenten innerhalb testen.
  • Warnung: @supports fallback definieren (z. B. display: block).

Kaskade & Layer

  • Merksatz: Spezifität gewinnt, bei Gleichstand entscheidet Source Order.
  • Übung: Layer-Hierarchie visualisieren & Prioritäten ändern.
  • Warnung: !important sparsam einsetzen, Inspector zeigt Alternativen.

MVP (Basis)

  • Sidebar mit Kategorien, Suche, Filter „Basics“.
  • Property-Panel mit Kern-Properties jeder Kategorie.
  • Code-Panel (Inline/Scoped/Global) & Copy-Button.
  • Live-Vorschau (Text, Card, Grid), Undo/Redo, Reset.
  • Preset-Auswahl (Flex-Center, 2-Spalten-Grid, Card-Schatten).
  • LocalStorage: letzte Session speichern.
  • Inspector mit Highlight & Konflikt-Hinweisen (Basis).

Erweiterungen Pro

  • Before/After-Split View & Visual Diff.
  • Specificity-Graph & Cascade-Layer-Editor.
  • Container-Query Playground mit multiplen Containern.
  • Animation Timeline & Scroll-Linked Animations.
  • Export als Zip (HTML, CSS, JSON Presets).
  • Team-Sharing via URL-Hash / JSON Export.
  • Guided Challenges & Schritt-für-Schritt-Lösungen.

13. Edge-Cases & Lernfallen

  • Shorthands vs. Longhands: UI warnt vor Überschreibung (z. B. margin setzt Einzelwerte zurück).
  • Unwirksame Werte: Tooltip erklärt „line-height: normal“ vs. numerischer Wert.
  • Stacking Context: Visual Layer zeigt neue Kontexte (transform, opacity < 1).
  • Containment & Container Queries: Zwei Container mit gleichem Component-Template zum Vergleich.
  • Fallbacks: Mehrere @font-face-Quellen, Background-Layer, Font-Stacks.
  • Cross-Property-Konflikte (z. B. overflow:hidden + Schatten) mit Warnhinweis.

14. Testfälle & Presets

  • Flex-Center: display:flex; justify-content:center; align-items:center;
  • 2-Spalten-Grid: grid-template-columns: repeat(2, 1fr); gap: 1rem;
  • Responsive Typo: font-size: clamp(1rem, 2vw + 0.5rem, 1.5rem);
  • Card-Schatten: border-radius: 12px; box-shadow: 0 8px 24px rgba(0,0,0,.15);
  • Overlay-Badge: position:absolute; inset:auto 8px 8px auto;
  • Hero Gradient Pro: Layered background mit radial & linear gradient.
  • Table Zebra: :nth-child(even) & background-color.
  • Animation Pulse: @keyframes + animation-iteration-count: infinite;
